## TL;DR
Game Design Document (GDD): A game design document is a communication artifact for game goals, rules, mechanics, content, constraints, and production decisions.

## Core Explanation
Good GDD practice is iterative. A document should help a team make, test, and revise a game rather than freeze every idea at the start. Books on game design emphasize communication, playcentric iteration, and production-stage clarity.
