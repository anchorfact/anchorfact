## 社区选择框架

### 按开发阶段选择

| 阶段 | 需求 | 推荐社区类型 |
|------|------|-------------|
| **学习入门** | 教程、问答、氛围友好 | 引擎官方社区、豆瓣小组、GDevelop |
| **原型开发** | 快速反馈、技术问题 | 引擎社区、Discord 频道 |
| **团队组建** | 找队友、找外包 | GameDevHub、Indie Gaming Studios |
| **发布营销** | 曝光、评测、玩家反馈 | itch.io、IndieDB |
| **职业发展** | 行业动态、海外机会 | igdshare、CGSociety、引擎世界 |

### 按职能选择

| 职能 | 核心需求 | 推荐社区 |
|------|----------|----------|
| **程序员** | 引擎技术、算法、性能 | 引擎官方论坛、GitHub Discussions |
| **美术/TA** | Shader、VFX、技术美术 | CGSociety、引擎世界、知乎 TA 话题 |
| **策划** | 设计理论、数据分析 | GDC 分享、igdshare、独立游戏小组 |
| **独立开发者** | 全栈、发布、商业 | itch.io、IndieDB、GameDevHub |
| **AI 开发者** | AI NPC、ML、工具链 | Player2、AI GameDev Lab |

## 社区参与策略

### 新手参与指南

| 步骤 | 行动 | 预期收益 |
|------|------|----------|
| **1. 潜水观察** | 浏览 2-4 周，了解社区文化 | 避免踩坑、找到优质内容源 |
| **2. 小问题起步** | 提出具体、可回答的技术问题 | 建立存在感、获得帮助 |
| **3. 分享经验** | 回复自己已解决的问题 | 回馈社区、建立声誉 |
| **4. 展示作品** | 发布开发日志、原型截图 | 获取反馈、吸引合作 |
| **5. 深度参与** | 组织活动、撰写教程 | 成为社区核心成员 |

### 跨社区信息流转

```
国际会议（GDC/CEDEC）
    ↓ 翻译整理
IGDShare 等中文社区
    ↓ 实践讨论
引擎社区（Cocos/UE/Unity）
    ↓ 工具化
GitHub / 开源项目
    ↓ 产品化
itch.io / Steam 发布
    ↓ 反馈回流
开发者日志 / 社区分享
```

## Game Jam 资源

### 主要 Game Jam 平台

| 平台/活动 | 周期 | 特色 | 链接 |
|-----------|------|------|------|
| **Global Game Jam** | 每年 1 月 | 全球最大线下+线上 Game Jam | globalgamejam.org |
| **Ludum Dare** | 每年 2-3 次 | 最老牌线上 Game Jam | ludumdare.com |
| **itch.io Game Jams** | 持续进行 | 主题多样、规模灵活 | itch.io/jams |
| **GMTK Game Jam** | 每年 1 次 | Mark Brown 主办、参与人数多 | itch.io |
| **Game Jam Plus** | 每年 1 次 | 有商业孵化后续 | gamejamplus.com |
| **国内 Game Jam** | 不定期 | CiGA、IndieAce 等组织 | 关注微博/B站 |

### Game Jam 开发策略

```
赛前准备（提前 1 周）：
├── 确定团队分工（程序/美术/策划/音效）
├── 准备通用代码框架
├── 准备素材库（音效、字体、UI 套件）
└── 确定技术栈（避免现场学习）

主题公布（第 1 小时）：
├── 头脑风暴（30 分钟）
├── 确定核心机制（1 个）
├── 制作最小可玩原型（4 小时）
└── 验证 fun hypothesis

开发期（剩余时间）：
├── 优先核心循环（必须可玩）
├── 次要高 polish（1-2 个亮点）
├── 最后打包测试（留 2-4 小时）
└── 绝不临时加功能

提交后：
├── 写开发日志（Post-mortem）
├── 收集玩家反馈
└── 评估是否继续开发
```

## 最佳实践

- [ ] **多社区并行**：不要只待在一个社区，不同社区有不同信息源
- [ ] **定期参与 Game Jam**：这是最快提升开发能力的方式
- [ ] **写开发日志**：记录过程既是分享也是自我梳理
- [ ] **回馈优先**：先帮助别人，再寻求帮助
- [ ] **关注会议转录**：GDC/CEDEC 干货是最高质量的学习资源
- [ ] **建立个人品牌**：持续输出内容，让社区认识你
- [ ] **线下线上结合**：线上交流 + 线下聚会（如 IGDShare 实体活动）
