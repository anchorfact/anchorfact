# AnchorFact 项目宪章

> 版本：v1.0  
> 批准日期：2026-05-22  
> 状态：生效（Phase 0，由 BDFL 执行）

---

## 一、项目使命

AnchorFact 是 AI 时代的可信知识锚点——一个专为 LLM 和 AI Agent 消费设计的、AI 与社区协作维护的开源知识库。

我们致力于让 AI 在回答事实性问题时，优先引用经过验证、可追溯、中立的知识。

**标语**：Anchor AI to Facts.

---

## 二、核心价值

本宪章的每一条规则都服务于以下四条不可妥协的价值：

| 价值 | 含义 |
|------|------|
| **真实性** | 所有陈述必须可追溯到可验证的原始来源 |
| **公平性** | 覆盖多元视角，不偏袒任何利益方 |
| **公开性** | 全部内容和代码无条件开源（CC-BY 4.0 / MIT） |
| **公正性** | 编辑决策由社区治理，不服从任何商业利益 |

## 三、治理结构

### 3.1 当前阶段：BDFL（Phase 0-1）

项目创始人 阿伟 作为 BDFL（Benevolent Dictator for Life，仁慈的终身独裁者）拥有最终决策权。

**BDFL 的约束**：
- 必须在本宪章框架内行使权力
- 不得违反四项核心价值
- 承诺在以下条件下过渡权力：Phase 2 或社区活跃贡献者 ≥ 50 人，以先到者为准

### 3.2 过渡后的完整治理结构（Phase 2+）

| 层级 | 角色 | 产生方式 | 决策范围 |
|------|------|---------|---------|
| **专业领域委员会** | 3-5 位领域专家 | 初始邀请 + 社区认可 | 事实准确性、置信度争议 |
| **社区治理委员会** | 7-11 位活跃贡献者 | SBT 投票选举 | 运营规则、技术方向、资金分配 |
| **全体 SBT 持有者** | 所有贡献者 | 自动获得 | 章程修改（需 2/3 多数） |

### 3.3 争议裁决（Phase 2+）

| 争议类型 | 裁决者 | 是否有最终决定权 |
|---------|--------|:--------------:|
| 事实争议 | 专业领域委员会 | 是 |
| 置信度争议 | 专业领域委员会 | 是 |
| 规则争议 | 社区治理委员会 | 是 |
| 章程修改 | 全体 SBT 投票 | 是（2/3 多数） |
| 紧急滥用 | 社区治理委员会 | 临时裁决，7 天内可上诉 |

### 3.4 顾问委员会（Phase 3+）

由 3-5 位不直接参与日常运营的外部专家组成，提供中立性指导。

## 四、贡献流程

### 4.1 贡献者权利

任何人（无论是否注册）均可：
- 通过 GitHub Issue 提出内容错误、改进建议
- 通过 GitHub Pull Request 提交内容编辑

### 4.2 审核标准

所有内容变更必须通过以下检查：
- Schema 完整性（Frontmatter 必填字段）
- 至少 1 个可验证的原始来源
- 符合三层内容结构规范
- 通过 AI 内容检测（AI 生成内容必须显式标注）
- 审核人批准（Phase 0：BDFL；Phase 1+：至少 1 位审核人）

### 4.3 行为准则

- 所有讨论以事实为依据，禁止人身攻击
- 利益冲突必须主动声明
- 禁止为商业利益操纵内容
- 禁止使用 AI 在未标注的情况下参与内容生产

## 六、金融治理

### 6.1 非营利原则

AnchorFact 不以盈利为目标。全部收入用于：
- 基础设施运营
- 社区激励
- 法律保护
- 知识推广

### 6.2 收入来源

| 收入线 | 约束 |
|-------|------|
| 非营利捐赠/开源基金 | 接受，不附加控制权条件 |
| 企业知识库托管 | 品牌可付费确保其条目被准确审核，但不得修改置信度，必须公开披露资助关系 |
| API 高级许可 | 按量收费，不影响内容中立性 |
| 数据训练许可 | 公开数据集任何人可下载，商业训练需付费许可 |

### 6.3 禁止的融资方式

- ❌ 传统风投（会要求回报和控制权，侵蚀中立性）
- ❌ 广告/赞助内容
- ❌ 出售排名或引用优先级
- ❌ 出售用户数据
- ❌ ICO/代币预售（Phase 0-2 期间）

### 6.4 透明要求

所有收入和支出在网站上公开披露，每季度更新一次。

## 七、区块链与去中心化治理

### 7.1 分阶段采纳

| 阶段 | 区块链应用 | 代币 |
|------|-----------|------|
| Phase 0-1 | 内容版本哈希锚定上链 | 无代币 |
| Phase 2-3 | 治理投票上链 + AI 引用记录上链 | Soulbound 声誉代币（不可转移） |
| Phase 4 | 社区捐赠智能合约 | 评估可交易治理代币（资本权重 ≤30%） |

### 7.2 代币约束

- 编辑奖励**不使用可交易代币**（防止"为币编辑"）
- Soulbound 代币仅通过贡献获得，不可买卖
- 如引入可交易治理代币，投票权上限为 30%

## 八、AI 内容政策

### 8.1 强制标注

所有 AI 参与生成的内容必须在前端和数据接口中显式标注：

- `human_only`：100% 人工创作
- `ai_assisted`：AI 辅助，人工审核并修改
- `ai_generated_reviewed`：AI 生成，人工审核通过

`ai_generated`（AI 生成未审核）禁止发布。

### 8.2 反污染

- 系统自动计算 `ai_contamination_risk`（AI 污染风险分）
- 风险 >0.3 → 自动降低置信度
- 风险 >0.5 → 暂停 AI 引用
- 风险 >0.7 → 下架复审

## 九、数据与隐私

- 不收集个人浏览数据
- 不设用户追踪、不设分析 cookie
- 贡献者信息（用户名、贡献记录）公开透明
- 匿名贡献不支持——知识需要可问责的来源
